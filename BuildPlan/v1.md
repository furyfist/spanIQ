# spanIQ V1 — Build Plan

> **One-liner:** A deterministic, zero-cost LLM evaluation framework that replaces LLM-as-judge with statistical detectors — same pytest interface as deepeval, zero API calls, fully reproducible scores.

**Author:** Himanshu (furyfist)
**Started:** June 2026
**Target completion:** 2–3 weeks
**Prior art:** DriftGuard v3 detectors (PSI, KS, JS) — proven, tested, mathematically correct

---

## The Problem We're Solving

Every team evaluating LLM outputs today uses LLM-as-judge — deepeval, ragas, braintrust all call GPT-4o/Claude to score outputs. Three compounding problems:

| Problem | Impact |
|---------|--------|
| **Cost** | 1000 test cases × 3 metrics × 3 LLM calls/metric = 9,000 API calls = $15–50/run. Daily CI = $500+/month just for evals |
| **Non-determinism** | Same input → different score across runs. Regression testing is unreliable. You can't know if a prompt change helped or if the judge was noisy |
| **No production monitoring** | deepeval/ragas are offline-only. Once your LLM is live, you're blind until a user complains |

spanIQ replaces the LLM judge with statistical detectors: PSI, KS test, JS divergence, embedding cosine similarity. Same pytest interface. $0 cost. Deterministic. Works offline and in production.

---

## What V1 Ships (Nothing More)

1. **`BaseMetric` abstract class** — interface-compatible with deepeval's `BaseMetric`
2. **4 deterministic metrics** — `ResponseDriftMetric`, `SemanticSimilarityMetric`, `OutputStabilityMetric`, `ConsistencyMetric`
3. **`LLMTestCase` data model** — compatible with deepeval's test case structure, extended with `baseline_outputs`
4. **`evaluate()` function** — runs metrics on test cases in parallel, prints cost comparison report
5. **pytest plugin** — `spaniq test run` CLI command
6. **Comprehensive test suite** — 40+ tests, all deterministic

## What V1 Does NOT Ship

- No dashboard / UI
- No cloud platform / hosted service
- No LLM integration of any kind
- No production monitoring (v2)
- No agent-specific metrics (v2)
- No RAG-specific metrics (v2)
- No CLI beyond `spaniq test run`

## Architecture

```
User Code
    │
    ▼
LLMTestCase(input, actual_output, expected_output, baseline_outputs)
    │
    ▼
evaluate([test_cases], [metrics])
    │
    ├─► ResponseDriftMetric.measure(test_case)
    │     └── PSI on token frequency distribution vs baseline
    │
    ├─► SemanticSimilarityMetric.measure(test_case)
    │     └── cosine similarity via sentence-transformers (local model, no API)
    │
    ├─► OutputStabilityMetric.measure(test_case)
    │     └── JS divergence on response length + structural features
    │
    └─► ConsistencyMetric.measure(test_case)
          └── KS test on embedding distance distribution across runs
    │
    ▼
EvalResult
    ├── per-metric scores (deterministic, reproducible)
    ├── pass/fail per test case
    └── cost comparison report ($0 vs estimated LLM-judge cost)
```

### Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| **sentence-transformers `all-MiniLM-L6-v2` for embeddings** | 80MB model, runs locally on CPU in <100ms per sentence, no API needed. Good enough for similarity — we're measuring drift, not generating text |
| **No LLM calls anywhere** | Core thesis. If we call an LLM, we're just deepeval with extra steps |
| **`baseline_outputs` as a first-class field** | This is the key insight — you need a reference distribution, not a reference answer. 10-50 baseline outputs define "what normal looks like" |
| **scipy for all statistical tests** | Proven, fast, well-tested. PSI is hand-implemented (industry-standard formula). KS and JS use scipy directly |
| **Parallel eval via `concurrent.futures`** | No async complexity. Thread pool for I/O (embedding computation). ProcessPool not needed — numpy/scipy release the GIL |
| **uv for project management** | 2026 standard. Replaces pip, venv, poetry. Single tool. Fast. Lock file for reproducibility |
| **ruff for linting + formatting** | Replaces black + flake8 + isort. Single tool. Fast. Configured in pyproject.toml |
| **hatchling as build backend** | Simple, fast, well-supported. No setup.py needed |

### Phase 1 — Statistical Core (Days 2–3)

**Goal:** Port DriftGuard's proven detectors into spaniq's `statistical/` package. These are the mathematical primitives everything else builds on.

**Step 1.1 — PSI detector (`src/spaniq/statistical/psi.py`)**

Port from `driftguard/detectors/psi.py`. Key decisions to preserve:

- Bin edges defined on baseline only (correct statistical practice)
- `np.percentile` binning for skewed distributions
- `np.unique()` on bin edges to prevent duplicate bins
- Epsilon smoothing (`1e-6`) to prevent `log(0)`
- Function signature: `compute_psi(baseline: np.ndarray, current: np.ndarray, n_bins: int = 10) → float`

This is a pure function now, not a class — no detector/severity logic. Just math.

**Step 1.2 — KS test wrapper (`src/spaniq/statistical/ks.py`)**

Port from `driftguard/detectors/ks_test.py`:

- Wraps `scipy.stats.ks_2samp`
- Returns `KSResult(statistic, p_value, significant)` — a named tuple
- Explicit `bool()` cast on `significant` (the `np.bool_` bug you already caught)
- Function signature: `compute_ks(baseline: np.ndarray, current: np.ndarray, alpha: float = 0.05) → KSResult`

**Step 1.3 — JS divergence wrapper (`src/spaniq/statistical/js.py`)**

Port from `driftguard/detectors/js_divergence.py`:

- Wraps `scipy.spatial.distance.jensenshannon`
- Shared bin edges across both arrays (`np.linspace`)
- Epsilon smoothing on histograms
- Returns JS distance (square root of divergence), bounded [0, 1]
- Function signature: `compute_js(baseline: np.ndarray, current: np.ndarray, n_bins: int = 50) → float`

**Step 1.4 — Embedding wrapper (`src/spaniq/statistical/embeddings.py`)**

New code. This is the bridge between text and statistical detectors.

```python
"""Lazy-loaded sentence-transformers wrapper.

The model is loaded once on first call and cached for the process lifetime.
Uses all-MiniLM-L6-v2: 80MB, runs on CPU, <100ms per sentence.
Deterministic: same input always produces same embedding.
"""

_model = None
MODEL_NAME = "sentence-transformers/all-MiniLM-L6-v2"

def get_model():
    global _model
    if _model is None:
        from sentence_transformers import SentenceTransformer
        _model = SentenceTransformer(MODEL_NAME)
    return _model

def embed(texts: list[str]) -> np.ndarray:
    """Encode texts to normalized embeddings. Returns (n, 384) array."""
    model = get_model()
    return model.encode(texts, normalize_embeddings=True, show_progress_bar=False)

def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    """Cosine similarity between two embedding vectors."""
    return float(np.dot(a, b))  # already normalized, so dot = cosine
```

Key decision — **lazy loading**: the model is 80MB and takes ~2s to load. We don't want to pay this cost if the user only uses non-embedding metrics. Import only triggers load on first `embed()` call.

### Phase 2 — Data Model (Day 4)

**Step 2.1 — `LLMTestCase` (`src/spaniq/core/test_case.py`)**

```python
from dataclasses import dataclass, field

@dataclass
class LLMTestCase:
    """A single evaluation test case.

    Compatible with deepeval's LLMTestCase interface, extended with
    baseline_outputs for distribution-based evaluation.

    Args:
        input: The prompt / query sent to the LLM
        actual_output: The LLM's response
        expected_output: Optional ground truth (for reference-based metrics)
        baseline_outputs: List of known-good outputs defining "normal" behavior.
                         This is what makes statistical evaluation possible —
                         instead of asking an LLM "is this good?", we compare
                         the actual_output's statistical properties against
                         the distribution of baseline_outputs.
        retrieval_context: Optional retrieved documents (for future RAG metrics)
        metadata: Optional dict for tagging (model version, prompt version, etc.)
    """
    input: str
    actual_output: str
    expected_output: str | None = None
    baseline_outputs: list[str] = field(default_factory=list)
    retrieval_context: list[str] | None = None
    metadata: dict | None = None

    def __post_init__(self):
        if not self.input:
            raise ValueError("input cannot be empty")
        if not self.actual_output:
            raise ValueError("actual_output cannot be empty")
```

**Why `baseline_outputs` is the key insight:**

deepeval asks: "is this output good?" (requires LLM judgment)
spaniq asks: "is this output statistically different from known-good outputs?" (requires math)

The baseline is a list of 10-50 outputs from your LLM when it was working correctly. You collect these once. Every future eval compares against this distribution.

### Phase 3 — Metrics (Days 5–8)

**Goal:** Implement 4 metrics. Each inherits `BaseMetric`, uses the statistical primitives from Phase 1. Zero LLM calls.

**Step 3.1 — `BaseMetric` (`src/spaniq/metrics/base.py`)**

```python
from abc import ABC, abstractmethod

class BaseMetric(ABC):
    """Abstract base for all spanIQ metrics.

    Interface-compatible with deepeval's BaseMetric.
    Subclasses implement measure() with pure statistical logic.
    """

    def __init__(self, threshold: float, name: str | None = None):
        self.threshold = threshold
        self.score: float | None = None
        self.reason: str = ""
        self.name = name or self.__class__.__name__

    @abstractmethod
    def measure(self, test_case: LLMTestCase) -> float:
        """Compute metric score. Must be deterministic.

        Returns:
            score: float — interpretation depends on metric.
                   Lower is better for drift/divergence metrics.
                   Higher is better for similarity metrics.
        """
        ...

    def is_successful(self) -> bool:
        """Whether the test case passed this metric."""
        if self.score is None:
            raise ValueError("measure() must be called before is_successful()")
        return self._check_threshold(self.score)

    @abstractmethod
    def _check_threshold(self, score: float) -> bool:
        """Define pass/fail logic. Overridden per metric."""
        ...
```

**Step 3.2 — `ResponseDriftMetric` (`src/spaniq/metrics/response_drift.py`)**

**What it measures:** Has the character of LLM responses shifted from baseline?

**How it works:**
1. Tokenize `actual_output` into character-level or word-level frequency distribution
2. Tokenize all `baseline_outputs` into aggregate frequency distribution
3. Run PSI between the two distributions
4. PSI < threshold → pass, PSI ≥ threshold → fail

**When it's useful:** Detecting prompt drift, model swaps, or degradation that changes the vocabulary/style of responses without necessarily changing semantic meaning.

```python
class ResponseDriftMetric(BaseMetric):
    def __init__(self, threshold: float = 0.10, tokenizer: str = "word"):
        super().__init__(threshold=threshold)
        self.tokenizer = tokenizer  # "word" or "char"

    def measure(self, test_case: LLMTestCase) -> float:
        if not test_case.baseline_outputs:
            raise ValueError("ResponseDriftMetric requires baseline_outputs")

        current_dist = self._tokenize_to_distribution(test_case.actual_output)
        baseline_dist = self._tokenize_to_distribution(
            " ".join(test_case.baseline_outputs)
        )
        self.score = compute_psi(baseline_dist, current_dist)
        self.reason = self._build_reason()
        return self.score

    def _check_threshold(self, score: float) -> bool:
        return score < self.threshold  # lower PSI = less drift = pass
```

**Step 3.3 — `SemanticSimilarityMetric` (`src/spaniq/metrics/semantic_similarity.py`)**

**What it measures:** Is the actual output semantically close to expected/baseline outputs?

**How it works:**
1. Embed `actual_output` using MiniLM
2. Embed `expected_output` or mean of `baseline_outputs` embeddings
3. Compute cosine similarity
4. similarity ≥ threshold → pass

**When it's useful:** Replacing deepeval's `AnswerRelevancyMetric` for cases where you have reference outputs. Deterministic, free, runs locally.

```python
class SemanticSimilarityMetric(BaseMetric):
    def __init__(self, threshold: float = 0.7):
        super().__init__(threshold=threshold)

    def measure(self, test_case: LLMTestCase) -> float:
        # prefer expected_output, fall back to baseline mean
        reference = test_case.expected_output
        if reference:
            vecs = embed([test_case.actual_output, reference])
            self.score = cosine_similarity(vecs[0], vecs[1])
        elif test_case.baseline_outputs:
            vecs = embed([test_case.actual_output] + test_case.baseline_outputs)
            actual_vec = vecs[0]
            baseline_mean = vecs[1:].mean(axis=0)
            baseline_mean /= np.linalg.norm(baseline_mean)
            self.score = cosine_similarity(actual_vec, baseline_mean)
        else:
            raise ValueError("Need expected_output or baseline_outputs")

        self.reason = self._build_reason()
        return self.score

    def _check_threshold(self, score: float) -> bool:
        return score >= self.threshold  # higher similarity = pass
```

**Step 3.4 — `OutputStabilityMetric` (`src/spaniq/metrics/output_stability.py`)**

**What it measures:** Is the LLM producing structurally consistent outputs?

**How it works:**
1. Extract structural features from `actual_output`: response length (chars), word count, sentence count, avg word length
2. Extract same features from all `baseline_outputs`
3. Run JS divergence between feature distributions
4. JS < threshold → pass

**When it's useful:** Detecting when a model starts producing wildly different length responses, or switches from structured to unstructured output. No LLM needed — pure structural analysis.

```python
class OutputStabilityMetric(BaseMetric):
    def __init__(self, threshold: float = 0.15):
        super().__init__(threshold=threshold)

    def measure(self, test_case: LLMTestCase) -> float:
        if not test_case.baseline_outputs:
            raise ValueError("OutputStabilityMetric requires baseline_outputs")

        current_features = self._extract_features(test_case.actual_output)
        baseline_features = np.array([
            self._extract_features(out) for out in test_case.baseline_outputs
        ])

        # JS divergence on each feature dimension, take mean
        scores = []
        for i in range(current_features.shape[0]):
            current_arr = np.array([current_features[i]])
            baseline_arr = baseline_features[:, i]
            scores.append(compute_js(baseline_arr, current_arr))

        self.score = float(np.mean(scores))
        self.reason = self._build_reason()
        return self.score

    def _extract_features(self, text: str) -> np.ndarray:
        words = text.split()
        sentences = text.split('.')
        return np.array([
            len(text),                              # char count
            len(words),                             # word count
            len([s for s in sentences if s.strip()]),# sentence count
            np.mean([len(w) for w in words]) if words else 0,  # avg word length
        ])

    def _check_threshold(self, score: float) -> bool:
        return score < self.threshold  # lower divergence = more stable = pass
```

**Step 3.5 — `ConsistencyMetric` (`src/spaniq/metrics/consistency.py`)**

**What it measures:** Given the same input, are repeated outputs consistent with each other?

**How it works:**
1. Embed `actual_output` and all `baseline_outputs`
2. Compute pairwise cosine distances between `actual_output` and each baseline
3. Run KS test: is the distance distribution significantly different from baseline-to-baseline distances?
4. Not significant → pass (output is within normal variation)

**When it's useful:** Detecting when a model becomes erratic — responses to the same prompt vary more than they used to. The KS test catches distributional shape changes that mean-based metrics miss.

```python
class ConsistencyMetric(BaseMetric):
    def __init__(self, threshold: float = 0.05):
        super().__init__(threshold=threshold)

    def measure(self, test_case: LLMTestCase) -> float:
        if len(test_case.baseline_outputs) < 5:
            raise ValueError("ConsistencyMetric needs ≥5 baseline_outputs")

        all_texts = [test_case.actual_output] + test_case.baseline_outputs
        vecs = embed(all_texts)

        actual_vec = vecs[0]
        baseline_vecs = vecs[1:]

        # distances: actual to each baseline
        actual_distances = np.array([
            1 - cosine_similarity(actual_vec, bv) for bv in baseline_vecs
        ])

        # distances: baseline to baseline (upper triangle)
        n = len(baseline_vecs)
        baseline_distances = []
        for i in range(n):
            for j in range(i + 1, n):
                baseline_distances.append(
                    1 - cosine_similarity(baseline_vecs[i], baseline_vecs[j])
                )
        baseline_distances = np.array(baseline_distances)

        ks_result = compute_ks(baseline_distances, actual_distances)
        self.score = ks_result.statistic
        self.reason = self._build_reason(ks_result)
        return self.score

    def _check_threshold(self, score: float) -> bool:
        return score < self.threshold  # lower KS stat = more consistent = pass
```

### Phase 4 — Evaluate Pipeline + Cost Report (Days 9–10)

**Goal:** Wire metrics into a single `evaluate()` function that runs everything and prints a cost comparison.

**Step 4.1 — `evaluate()` function (`src/spaniq/core/evaluate.py`)**

```python
from concurrent.futures import ThreadPoolExecutor
import time

def evaluate(
    test_cases: list[LLMTestCase],
    metrics: list[BaseMetric],
    max_workers: int = 4,
    verbose: bool = True,
) -> EvalResult:
    """Run all metrics on all test cases. Parallel. Deterministic.

    This is the main entry point. Compatible with deepeval's evaluate() signature.
    """
    start = time.perf_counter()
    results = []

    with ThreadPoolExecutor(max_workers=max_workers) as pool:
        futures = []
        for tc in test_cases:
            for metric in metrics:
                # clone metric to avoid shared state
                m = _clone_metric(metric)
                futures.append(pool.submit(_run_single, m, tc))

        # collect results grouped by test case
        # ...

    duration = time.perf_counter() - start
    cost_report = _compute_cost_report(len(test_cases), len(metrics))

    eval_result = EvalResult(
        test_case_results=results,
        total_passed=sum(1 for r in results if r.passed),
        total_failed=sum(1 for r in results if not r.passed),
        total_cases=len(test_cases),
        cost_report=cost_report,
        duration_seconds=duration,
    )

    if verbose:
        print_report(eval_result)

    return eval_result
```

```python
GPT4O_OUTPUT_PER_1K = 0.01
AVG_TOKENS_PER_JUDGE_CALL = 800  # input + output

def estimate_llm_judge_cost(n_test_cases: int, n_metrics: int) -> float:
    """Estimate what this eval run would cost with LLM-as-judge."""
    calls_per_metric = 3  # decompose + judge + aggregate (deepeval pattern)
    total_calls = n_test_cases * n_metrics * calls_per_metric
    total_tokens = total_calls * AVG_TOKENS_PER_JUDGE_CALL
    input_cost = (total_tokens * 0.6) / 1000 * GPT4O_INPUT_PER_1K
    output_cost = (total_tokens * 0.4) / 1000 * GPT4O_OUTPUT_PER_1K
    return input_cost + output_cost
```

```python
# src/spaniq/__init__.py
def assert_eval(test_case: LLMTestCase, metrics: list[BaseMetric]) -> None:
    """Pytest-friendly assertion. Raises AssertionError on failure."""
    result = evaluate([test_case], metrics, verbose=False)
    if not result.test_case_results[0].passed:
        failures = [
            f"{mr.metric_name}: {mr.score:.4f} (threshold: {mr.threshold})"
            for mr in result.test_case_results[0].metric_results
            if not mr.passed
        ]
        raise AssertionError(f"Metrics failed: {', '.join(failures)}")
```

## Honest Limitations (Document These)

spanIQ does NOT replace LLM-as-judge for:

1. **Subjective quality** — "is this response helpful?" requires judgment, not statistics
2. **Factual accuracy** — PSI can't tell you if a claim is true, only if it's different from baseline
3. **Safety/toxicity** — requires semantic understanding, not distributional analysis
4. **Zero-shot evaluation** — if you have no baselines, statistical metrics have nothing to compare against
5. **Complex reasoning** — "did the model follow the chain of thought correctly?" needs an LLM

spanIQ IS effective for:
1. **Regression detection** — did this prompt change make outputs drift?
2. **Consistency monitoring** — is the model producing stable outputs?
3. **Semantic similarity** — is the output close to known-good responses?
4. **Production monitoring** — continuous, free, deterministic quality signal
5. **CI/CD gating** — reproducible pass/fail on every PR
